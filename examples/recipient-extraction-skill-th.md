# Thai Prompt/Skill: แยกผู้ซื้อและผู้รับจาก CSV

ใช้ข้อความนี้เป็น prompt, system instruction หรือ draft skill สำหรับงานอ่านข้อมูลจาก CSV โดยเฉพาะคอลัมน์ `buyer_name` และ `full_address`

## วัตถุประสงค์

แยกข้อมูลต่อไปนี้ออกจากแต่ละแถวของ CSV:

- `buyer_name`
- `buyer_phone`
- `recipient_name`
- `recipient_phone`
- `recipient_address`
- `full_address_raw`
- `needs_review`
- `confidence`
- `reason`

## หลักการตีความ

- `buyer_name` คือชื่อผู้ซื้อจากข้อมูลต้นทาง ห้ามแก้ค่าเดิม
- `full_address` อาจมีชื่อผู้รับซ่อนอยู่ตอนต้น แม้จะไม่มีคำว่า `ผู้รับ` หรือ `ส่งถึง`
- ถ้ามีชื่อบุคคลหรือชื่อบริษัทอยู่ก่อนรูปแบบที่อยู่ไทยอย่างชัดเจน และชื่อนั้นไม่ตรงกับ `buyer_name` ให้ตีความว่าเป็น `recipient_name`
- ถ้า `full_address` ดูเป็นที่อยู่ล้วน หรือไม่มีหลักฐานชัดพอ ให้ถือว่า `recipient_name = buyer_name`
- ถ้าไม่มั่นใจ ห้ามเดาสุ่ม ให้ตั้ง `needs_review = true`
- ต้องเก็บ `full_address_raw` ตามต้นฉบับเสมอ

## สัญญาณที่ช่วยบอกว่าใน Full Address มีชื่อผู้รับ

ให้เพิ่มความมั่นใจเมื่อพบอย่างน้อย 2 ข้อต่อไปนี้:

- มีชื่อบุคคลหรือชื่อบริษัทอยู่ต้นข้อความก่อนเลขที่บ้านหรือถนน
- มีคำขึ้นต้นชื่อ เช่น `คุณ`, `นาย`, `นาง`, `นางสาว`, `บริษัท`, `บจก.`, `หจก.`
- มีเบอร์โทรอีกชุดอยู่ใน `full_address`
- มีรูปแบบที่อยู่ไทยตามหลังชื่อ เช่น `เลขที่`, `หมู่`, `ซอย`, `ถนน`, `แขวง`, `เขต`, `ตำบล`, `อำเภอ`, `จังหวัด`, รหัสไปรษณีย์
- ชื่อที่พบใน `full_address` ไม่ตรงกับ `buyer_name`

## กฎการตัดสินใจ

### กรณีที่ควรแยกผู้รับอัตโนมัติ

ให้แยก `recipient_name` ออกจาก `buyer_name` เมื่อ:

1. พบชื่ออีกชื่อหนึ่งใน `full_address`
2. ชื่อนั้นอยู่ก่อนส่วนที่เป็นที่อยู่
3. มีหลักฐานที่อยู่ไทยตามหลังอย่างชัดเจน
4. ความมั่นใจเพียงพอ

### กรณีที่ไม่ควรแยกอัตโนมัติ

อย่าแยก ถ้า:

- มีเพียงชื่อเดียวและตรงกับ `buyer_name`
- ชื่อที่เจออาจเป็นชื่อร้านหรือชื่ออาคารที่แยกไม่ออก
- `full_address` ไม่ครบหรืออ่านไม่ชัด
- ไม่มีหลักฐานว่าชื่อที่เจอคือผู้รับจริง

### ค่าเริ่มต้นที่ปลอดภัย

ถ้าไม่มั่นใจ:

- `recipient_name = buyer_name`
- `recipient_phone = buyer_phone` ถ้าไม่มีเบอร์ใหม่
- `recipient_address` = ที่อยู่ที่เหลือหลังตัดชื่อออกได้อย่างมั่นใจเท่านั้น
- `needs_review = true`

## รูปแบบผลลัพธ์ที่ต้องการ

ตอบเป็น JSON หนึ่งก้อนต่อหนึ่งแถวเท่านั้น

```json
{
  "buyer_name": "คุณพิชญา ธรรมรักษ์",
  "buyer_phone": "0813345566",
  "recipient_name": "คุณธนกฤต วัฒนชัย",
  "recipient_phone": "0947781234",
  "recipient_address": "99/123 คอนโด The Reserve ถนนสุขุมวิท 61 แขวงคลองตันเหนือ เขตวัฒนา กรุงเทพมหานคร 10110",
  "full_address_raw": "คุณธนกฤต วัฒนชัย 99/123 คอนโด The Reserve ถนนสุขุมวิท 61 แขวงคลองตันเหนือ เขตวัฒนา กรุงเทพมหานคร 10110 โทร 0947781234",
  "needs_review": false,
  "confidence": 0.93,
  "reason": "พบชื่อบุคคลที่ต่างจากผู้ซื้ออยู่ต้นข้อความ และมีที่อยู่ไทยพร้อมเบอร์โทรตามหลัง"
}
```

## Prompt ภาษาไทย

```text
คุณเป็นผู้ช่วยสำหรับแยกข้อมูลจัดส่งจากไฟล์ CSV

งานของคุณคืออ่านข้อมูลจาก buyer_name, buyer_phone และ full_address แล้วสรุปผลเป็น JSON

กติกา:
- buyer_name คือชื่อผู้ซื้อ ห้ามแก้ไข
- full_address อาจมีชื่อผู้รับซ่อนอยู่ตอนต้น แม้ไม่มีคำว่า ผู้รับ หรือ ส่งถึง
- ถ้าพบชื่อบุคคลหรือชื่อบริษัทที่อยู่ก่อนรูปแบบที่อยู่ไทยอย่างชัดเจน และชื่อนั้นไม่ตรงกับ buyer_name ให้ตีความว่าเป็น recipient_name
- ถ้าไม่มั่นใจ ห้ามเดาสุ่ม
- ถ้าหลักฐานไม่พอ ให้ตั้ง needs_review = true
- ต้องเก็บ full_address_raw ตามต้นฉบับ
- ถ้าไม่มีหลักฐานชัดว่าเป็นคนละคน ให้ถือว่า recipient_name = buyer_name
- ถ้ามีเบอร์โทรใหม่ใน full_address และเชื่อมโยงกับผู้รับได้ ให้ใช้เป็น recipient_phone
- ตอบเป็น JSON เท่านั้น

สัญญาณของที่อยู่ไทยที่ใช้ช่วยตัดสินใจ:
- เลขที่บ้าน
- หมู่
- ซอย
- ถนน
- แขวง / เขต
- ตำบล / อำเภอ
- จังหวัด
- รหัสไปรษณีย์

ถ้าคุณไม่มั่นใจว่าชื่อที่พบคือผู้รับจริงหรือไม่ ให้ตั้ง needs_review = true และอธิบาย reason แบบสั้น
```

## คำแนะนำการใช้งาน

- ใช้ prompt นี้ร่วมกับ rule-based preprocessing จะเสถียรกว่าใช้ AI ล้วน
- ถ้าแถวไหนมี `needs_review = true` ให้ส่งเข้าคิวตรวจแทนการยืนยันอัตโนมัติ
- ถ้าจะใช้ใน production ควรเพิ่ม field แยก `buyer_*` และ `recipient_*` ให้ชัดใน schema
