# MoveVAI Shipping Schema Rules

เอกสารนี้ใช้เป็นกติกาอ้างอิงสำหรับแปลงข้อมูลคำสั่งซื้อไปเป็น schema ปลายทางของงานจัดส่ง

## แนวคิดหลัก

- ต้นทางอาจมีหลาย schema
- ให้ใช้ชื่อ header และเนื้อหาจริงเป็นตัว map
- ชื่อไฟล์เป็นเพียงตัวอย่าง ไม่ใช่กติกาของระบบ

## ฟิลด์ปลายทางที่สำคัญ

### ข้อมูลผู้รับ

- `RECEIVER`
  - ใช้ชื่อผู้รับที่สกัดได้จาก `full_address` ถ้ามั่นใจ
  - ถ้าไม่มั่นใจ ใช้ชื่อผู้ซื้อ

- `RECEIVER_ADDRESS`
  - ถ้า source มี `full_address` ให้ใช้ค่านั้นตรง ๆ เป็นค่าแรกสำหรับ `RECEIVER_ADDRESS`
  - ถ้าไม่มี `full_address` ให้ใช้ raw address field ที่ดีที่สุดจาก source ตรง ๆ
  - ห้ามตัด, ย่อ, normalize, rewrite, หรือเรียบเรียง address ใหม่
  - ห้ามเติม token ใหม่จาก field อื่น ถ้า token นั้นไม่มีอยู่จริงใน address ของผู้รับ
  - ถ้า `Address` กับ `Full Address` ขัดกัน ให้ยึดเฉพาะค่าที่ตรวจสอบย้อนกลับได้จาก source เดิม
  - ให้คงรายละเอียดทั้งหมดตาม source เดิม แม้จะมีบ้านเลขที่, หมู่บ้าน, อาคาร, ชั้น, ซอย, ถนน, หน้าร้าน, landmark, เขต/อำเภอ, จังหวัด, หรือรหัสไปรษณีย์ อยู่ใน field เดียวกัน

- `RECEIVER_AMPHUR`
  - ใช้ `District`
  - ถ้าเป็นกรุงเทพมหานคร ให้ format เป็น `เขต...`
  - ถ้าเป็นต่างจังหวัด ให้ format เป็น `อ....`

- `RECEIVER_PROVINCE`
  - ใช้ `Province` จากต้นทาง

- `RECEIVER_ZIPCODE`
  - ใช้ `Postal Code`

- `RECEIVER_TEL`
  - ใช้เบอร์ที่พบใน `full_address` ก่อน ถ้ามีและชัดว่าเป็นของผู้รับ
  - ถ้าไม่มี ให้ fallback ไป `Mobile Number`
  - ถ้ายังไม่มี ให้ fallback ไป `User Phone No`
  - format เป็นเบอร์ไทยอ่านง่ายเสมอ

## กฎแยกผู้รับจาก Full Address

ให้แยกชื่อผู้รับอัตโนมัติเมื่อพบเงื่อนไขร่วมกัน:

1. มีชื่อบุคคลหรือชื่อบริษัทอยู่ต้นข้อความ
2. ชื่อนั้นต่างจาก `buyer_name`
3. หลังชื่อนั้นมี pattern ที่อยู่ไทยชัดเจน เช่น
   - เลขที่บ้าน
   - หมู่
   - ซอย
   - ถนน
   - แขวง / เขต
   - ตำบล / อำเภอ
   - จังหวัด
   - รหัสไปรษณีย์

ตัวอย่าง:

- `คุณธนกฤต วัฒนชัย 99/123 ...` -> `RECEIVER = คุณธนกฤต วัฒนชัย`
- `ผู้รับ คุณชลธิชา สุวรรณดี อาคาร...` -> `RECEIVER = คุณชลธิชา สุวรรณดี`

ตัวอย่างสิ่งที่ห้ามทำ:

- ถ้า `Full Address` เป็น `อาคารเอสเอ็มเอ็ม ชั้น 9 ถนนเทพรัตน ...`
  และ `Address` อีก field มี `88 อาคารเอสเอ็มเอ็ม ชั้น 9 ถนนเทพรัตน`
  ห้ามนำ `88` ไปเติมใน `RECEIVER_ADDRESS` ถ้า `88` ไม่ได้อยู่ใน address ของผู้รับที่ยืนยันได้
- ถ้า address จริงคือ `1111/1111 2 บ้านใหม่เก่า เมืองปทุมธานี ปทุมธานี 12000`
  ให้ส่งค่า `RECEIVER_ADDRESS` ตรงตามนี้ทั้งก้อน
  ห้ามตัดให้เหลือแค่ `1111/1111 2`

## กรณีก้ำกึ่ง

ถ้าใน `full_address` มีทั้งชื่อคนและชื่อบริษัทติดกัน เช่น:

- `คุณณัฐวุฒิ เจริญผล บริษัท เอ็นซี โลจิสติกส์ ...`

ให้ใช้หลักนี้:

- ถ้าจำเป็นต้องส่งออกทันทีและยังไม่มี human review ให้คงค่าที่สกัดได้ทั้งก้อน แต่ควรแจ้งว่าเป็นเคสก้ำกึ่ง
- ถ้ามีขั้นตรวจทาน ให้ mark ไว้สำหรับ review เพราะอาจต้องตัดให้เหลือเฉพาะชื่อบุคคล

## กฎเบอร์โทร

- ล้างอักขระที่ไม่ใช่ตัวเลขก่อน
- ถ้าเป็น 10 หลักและขึ้นต้น `0` ให้ format เป็น `0XX-XXX-XXXX`
- ถ้าเป็น 9 หลักและขึ้นต้น `02` ให้ format เป็น `02-XXX-XXXX`
- ห้ามเติม quote (`'`) นำหน้าเบอร์

## ค่า default ที่ใช้ได้เมื่อผู้ใช้ไม่ได้กำหนด

- `INSURE_PRICE = 1` ถ้าผู้ใช้ระบุชัดว่าต้องการค่านี้ใน schema ปลายทาง
- สำหรับ MoveVAI shipping CSV ให้ใช้ค่า fix ดังนี้:
  `PRODUCT_IN_BOX = ลำดับแถวแบบ 1-based`
  `WEIGHT = 0`
  `PRICE = 0`
- `INSURE` และ field อื่นที่เป็นระบบขนส่ง ให้ยึดตาม template ปลายทางหรือคำสั่งล่าสุดของผู้ใช้

## field ที่ต้องเว้นว่างเสมอ (ห้ามใส่ค่าใด ๆ)

field ต่อไปนี้ให้ export เป็นค่าว่างเสมอ ไม่ว่า source จะมีข้อมูลหรือไม่:

- `COD_DETAIL_NAME`
- `COD_DETAIL_SIZE`
- `COD_DETAIL_VOLUME`
- `COD_DETAIL_QTY`
- `COD_DETAIL_COLOR`
- `COD_DETAIL_QTY_AMOUNT`
- `PROVE_OF_PAYMENT`
- `IS_CONSENT`

ห้ามนำค่าจาก source มาใส่ใน field เหล่านี้ แม้จะมีข้อมูลที่ดูเหมือน map ได้ก็ตาม

## วิธีคิดเวลาแปลงไฟล์

1. อ่าน header ของ source
2. อ่าน header ของ target
3. สรุป mapping field ต่อ field
4. แปลงข้อมูลด้วย rule ที่อธิบายได้
5. ตรวจตัวอย่างแถวปกติและแถวพิเศษเสมอ
